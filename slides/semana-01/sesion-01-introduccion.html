IA, Machine Learning y Deep Learning

Inteligencia artificial
└── Aprendizaje automático
    └── Aprendizaje profundo
  • IA: sistemas que realizan tareas asociadas con comportamiento inteligente.
  • ML: sistemas que ajustan su comportamiento usando datos.
  • DL: ML basado en composiciones de capas que aprenden representaciones.

Deep Learning no es sinónimo de IA generativa.

El cambio central: aprender representaciones

En modelos clásicos solemos diseñar muchas variables de entrada.

En una red profunda, las capas intermedias pueden construir representaciones progresivas:

Dato→Patrones simples→Representación→Decisión

Profundidad no significa magia

Una red sigue siendo una función parametrizada:

\hat y=f(x;\theta)

  • x: dato de entrada;
  • \theta: pesos y sesgos aprendibles;
  • f: composición definida por la arquitectura;
  • \hat y: salida del modelo.

La arquitectura restringe qué clase de funciones resulta fácil aprender.

Abramos una capa

Una capa densa calcula Z=XW+b y después aplica una función A=\phi(Z).

Objeto Shape de ejemplo Papel
X (32, 10) 32 casos, 10 características
W (10, 8) conexiones aprendibles
b (8,) desplazamiento por unidad
A (32, 8) nueva representación

No conocemos aún los valores de W, pero ya podemos auditar la operación.

Elegir arquitectura es formular una hipótesis sobre la estructura del problema.

Ejemplo: visión

Una CNN no recibe el concepto “borde”. Aprende filtros reutilizables que responden a configuraciones espaciales.

Si barajamos todos los píxeles de una imagen de la misma forma, ¿qué información se conserva y cuál se destruye?

Ejemplo: lenguaje

La frase cambia si cambia el orden. Una representación útil debe distinguir:

  • qué elementos aparecen;
  • dónde aparecen;
  • con cuáles otros se relacionan;
  • qué contexto modifica su significado.

La atención hará explícita parte de esa interacción, pero no convierte automáticamente una explicación en causalidad.

Ejemplo: series de tiempo

Predecir demanda mañana no permite mezclar observaciones futuras durante la validación.

Deep Learning no elimina las reglas metodológicas aprendidas:

  • particiones honestas;
  • baseline;
  • métrica ligada al costo;
  • trazabilidad;
  • análisis de error.

Lo que puede salir mal

  • La pérdida no disminuye.
  • El gradiente se desvanece o explota.
  • La red memoriza el entrenamiento.
  • La métrica oculta fallas importantes.
  • El modelo aprende un artefacto del dataset.
  • El costo de entrenar no justifica la mejora.

Este curso trata esos comportamientos como evidencia para diagnosticar.

Cómo trabajaremos

Cada mecanismo seguirá el mismo recorrido:

  1. intuición;
  2. operación matemática y shapes;
  3. código mínimo;
  4. experimento controlado;
  5. visualización;
  6. interpretación y límites.

Proyecto: una pregunta incómoda

¿La profundidad realmente se justificaba para este problema?

El proyecto debe:

  • comprender y replicar un artículo;
  • trasladarlo a un problema local;
  • compararlo con una línea base;
  • auditar qué aprendió;
  • dictaminar con evidencia, incluso si la respuesta es “no”.

Tres fases

Fase Evidencia
Comprensión y formulación artículo, problema, datos, baseline y plan
Replicación y traslado reproducción, transferencia y dinámica del entrenamiento
Auditoría y comunicación explicabilidad, fallas, costo y dictamen

Todos los integrantes deben poder defender las decisiones técnicas.

Reglas de reproducibilidad

  • Semillas y dependencias declaradas.
  • Separación de selección y evaluación final.
  • Origen y licencia de datos documentados.
  • Configuración y curvas conservadas.
  • Uso de IA declarado y comprendido.
  • Resultados negativos reportados, no escondidos.
