El error que queremos evitar

ValueError: expected shape=(None, 10), found shape=(10, 32)

La mayoría de los primeros errores de una red no son “de IA”: son desacuerdos entre la semántica de los ejes.

De escalar a tensor

Objeto Rango Ejemplo
Escalar 0 temperatura
Vector 1 características de un caso
Matriz 2 lote tabular
Tensor 3+ secuencia, imagen o lote estructurado

El rango cuenta ejes; el shape informa su longitud.

Shape no es significado

Dos tensores pueden tener shape (32, 10) y representar cosas distintas:

  • 32 clientes con 10 variables;
  • 32 secuencias con 10 pasos;
  • 32 palabras representadas por 10 números.

Siempre nombre la semántica de cada eje antes de diseñar la capa.

El eje batch

En datos tabulares:

X\in\mathbb{R}^{B\times D}

  • B: número de observaciones procesadas juntas;
  • D: características por observación.

Keras suele expresar el batch desconocido como None: (None, D).

Una capa densa

Para producir H unidades:

Z=XW+b

Objeto Shape
X (B, D)
W (D, H)
b (H,)
Z (B, H)

El broadcasting suma el mismo vector b a cada fila.

¿Cuántos parámetros aprende?

\underbrace{D\times H}_{pesos}+\underbrace{H}_{sesgos}

Una capa Dense(4) que recibe 3 características aprende:

3\times4+4=16

El batch no cambia el número de parámetros.

  2. ¿Cuántos parámetros se aprenden?

De una capa a un MLP

A^{(1)}=\phi(XW^{(1)}+b^{(1)})

\hat y=\sigma(A^{(1)}W^{(2)}+b^{(2)})

Cada salida de capa se convierte en la entrada de la siguiente.

¿Por qué necesitamos activación?

Sin activaciones no lineales:

(XW_1)W_2=X(W_1W_2)

Apilar transformaciones lineales seguiría siendo una sola transformación lineal. ReLU, sigmoid y otras activaciones cambian la familia de funciones representables.

Microred 2–2–1

Usaremos:

x: 2 características
oculta: 2 unidades ReLU
salida: 1 unidad sigmoid

Parámetros:

2\times2+2+2\times1+1=9

Abramos el forward pass

  1. Z_1=XW_1+b_1
  2. A_1=ReLU(Z_1)
  3. Z_2=A_1W_2+b_2
  4. \hat y=sigmoid(Z_2)

Cada paso conservará un valor, un shape y una interpretación.

NumPy: operación explícita

z1 = X @ W1 + b1
a1 = np.maximum(z1, 0.0)
z2 = a1 @ W2 + b2
y_hat = 1 / (1 + np.exp(-z2))

No hay una red escondida: hay álgebra lineal, no linealidad y parámetros.

Keras: la misma operación

model = keras.Sequential([
    keras.Input(shape=(2,)),
    keras.layers.Dense(2, activation="relu"),
    keras.layers.Dense(1, activation="sigmoid"),
])

Asignaremos los mismos pesos y verificaremos np.allclose(y_numpy, y_keras).

Keras todavía no aprendió

Construir define la función. Para aprender faltan:

  • una función de pérdida;
  • un optimizador;
  • ejemplos con respuestas;
  • un ciclo de actualización.

compile() configura el aprendizaje; fit() ejecuta actualizaciones.

Un problema no lineal pequeño

Usaremos datos de dos medias lunas:

  • dos características visibles;
  • dos clases;
  • frontera no lineal;
  • entrenamiento rápido en CPU;
  • comparación con regresión logística.

El objetivo no es “ganar”, sino observar qué representación necesita el problema.

La capa de salida depende de la tarea

Tarea Unidades Activación Pérdida típica
Regresión 1 o más lineal MSE/MAE
Binaria 1 sigmoid binary cross-entropy
Multiclase exclusiva K softmax categorical cross-entropy

La salida expresa la estructura de la variable objetivo.

Compilar el primer MLP

model.compile(
    optimizer=keras.optimizers.SGD(learning_rate=0.05),
    loss="binary_crossentropy",
    metrics=["accuracy"],
)

Usamos SGD visible y simple. Adam llegará cuando podamos comparar su comportamiento con criterio.

¿Qué observamos durante fit?

history = model.fit(
    X_train, y_train,
    validation_data=(X_val, y_val),
    epochs=80,
    batch_size=32,
    verbose=0,
)

history es evidencia: pérdida y métricas por época, separadas para entrenamiento y validación.

Una primera lectura de curvas

  • Ambas pérdidas bajan: hay evidencia de aprendizaje.
  • Entrenamiento baja y validación sube: posible sobreajuste.
  • Ninguna baja: capacidad, datos, optimizador o tasa requieren revisión.
  • Oscilaciones fuertes: el proceso puede ser inestable.

Todavía no diagnosticamos causas; aprendemos a formular preguntas.

Baseline antes de profundidad

La pregunta correcta no es “¿qué accuracy logró la red?”, sino “¿qué aporta frente a una referencia y a qué costo?”.

Errores frecuentes

  • Transponer ejemplos y características.
  • Confundir rango con shape.
  • Contar el batch como parámetro.
  • Usar activación/pérdida incompatibles.
  • Evaluar repetidamente sobre test.
  • interpretar model.summary() sin comprobar las operaciones.
